# Parcel Delivery System

## Use case
Створення відправлення з перевіркою параметрів і розрахунком ціни.

## Архітектура
Hexagonal Architecture

## Чому обрана
Дозволяє відокремити бізнес-логіку від технічних деталей.
Система легко тестується та розширюється.

## Структура
- domain — бізнес-логіка
- application — use case
- port — інтерфейс
- adapter — реалізація
- main — запуск

## Розподіл відповідальностей
- Domain: правила перевірки і ціна
- Use case: координація
- Ports: контракт
- Adapters: реалізація збереження

## Запуск
Run App.java
